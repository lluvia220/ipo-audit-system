# IPO审核动态查询系统使用指南

## 系统概述

IPO审核动态查询系统是一个专业的IPO审核数据分析与报告平台，提供月度专项报告、周报统计和案例数据库三大核心功能。

## 主要功能

### 📊 月度专项报告
- **文字深度分析**: 提供详细的IPO审核趋势分析
- **图表辅助展示**: 使用ECharts生成直观的统计图表
- **趋势分析**: 展示审核通过率、拒绝率等关键指标的变化趋势

### 📈 周报统计
- **关键词统计**: 自动识别和分析IPO审核中的高频关键词
- **图表为主展示**: 使用饼图、柱状图等多种图表形式
- **简单文字说明**: 提供简要的周度分析报告

### 📋 案例数据库
- **真实案例**: 包含完整的IPO审核案例信息
- **检索功能**: 支持公司名称、关键词、行业、状态等多维度检索
- **详细分析**: 每个案例包含完整的审核过程和结果信息

## 系统要求

- **Node.js**: >= 14.0.0
- **Python3**: >= 3.6
- **浏览器**: 现代浏览器 (Chrome, Firefox, Safari, Edge)

## 安装与启动

### 1. 下载项目
```bash
git clone <项目地址>
cd ipo-audit-system
```

### 2. 一键启动
```bash
./start.sh
```

启动脚本会自动：
- 安装后端依赖
- 启动后端API服务 (端口3000)
- 启动前端Web服务 (端口8080)
- 检查服务运行状态

### 3. 访问系统
- **前端界面**: http://localhost:8080
- **后端API**: http://localhost:3000
- **健康检查**: http://localhost:3000/health

### 4. 停止服务
```bash
./stop.sh
```

## 使用说明

### 月度专项报告
1. 进入"月度专项报告"标签页
2. 查看不同月份的审核统计和分析
3. 使用搜索框查找特定月份或关键词
4. 查看详细的柱状图统计

### 周报统计
1. 进入"周报统计"标签页
2. 查看每周的申请数量统计
3. 观察关键词分布的饼图
4. 阅读简要的周度分析

### 案例数据库
1. 进入"案例数据库"标签页
2. 使用搜索框搜索公司名称或关键词
3. 使用筛选器按行业或审核状态过滤
4. 查看每个案例的详细信息

## API接口

### 健康检查
```
GET /health
```

### 获取所有案例
```
GET /api/cases
查询参数:
- search: 搜索关键词
- industry: 行业筛选
- status: 状态筛选
```

### 获取月度报告
```
GET /api/monthly-reports
```

### 获取周报
```
GET /api/weekly-reports
```

### 搜索案例
```
GET /api/search-cases?keyword=关键词
```

### 获取案例详情
```
GET /api/cases/:id
```

## 故障排除

### 端口占用
如果端口3000或8080被占用，请：
1. 使用`lsof -ti:端口号`查看占用进程
2. 使用`kill 进程ID`终止进程
3. 重新启动服务

### 依赖安装失败
如果Node.js依赖安装失败，请：
1. 检查网络连接
2. 清除npm缓存: `npm cache clean --force`
3. 重新运行安装命令

### 前端无法访问
如果前端无法访问：
1. 检查Python3是否安装: `python3 --version`
2. 确认8080端口未被占用
3. 查看frontend.log日志文件

## 技术支持

如遇到技术问题，请查看日志文件：
- `backend.log`: 后端服务日志
- `frontend.log`: 前端服务日志

或联系开发团队获取支持。
